PAYMENT INTEGRATION RULE — MANDATORY

For all generated customer applications, PayPal is the ONLY permitted default payment provider.

REQUIRED PayPal integration pattern:
  - PayPal Checkout orders for one-time payments
  - PayPal Subscriptions for recurring plans
  - PayPal Webhooks for lifecycle events
  - PayPal client ID only on the frontend; all secret API calls stay server-side
  - All PayPal keys from environment variables ONLY:
      PAYPAL_CLIENT_ID (client-safe)
      PAYPAL_CLIENT_SECRET (server-side, never expose)
      PAYPAL_MODE (sandbox or live)

FORBIDDEN:
  - Any Braintree import, require, or service call
  - Any Stripe package, Stripe.js, STRIPE_* env var, or stripe.com API call
  - Hardcoded payment keys of any kind

This rule is enforced by the Build Integrity Validator (PAYPAL_ONLY_VIOLATION error).
Any Stripe or Braintree code in a generated customer app will cause job failure and require repair.
