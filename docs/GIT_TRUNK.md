# Git remotes and trunk (operator note)

**Compliance §4.9** — This is **policy and process**, not something the app can “complete.”

- The repo may have both `origin` and `crucibai` (or other) remotes; **one trunk** (main branch) should be agreed by the team.
- **Merge / rebase / push** after verifying local work — **do not push** until the product owner approves (see `PHASE0_COMPLIANCE_AND_PROOF.md` P0-02).

No automated proof applies; execution is **by you** after verification.
